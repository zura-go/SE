01 · იდენტობა

Software Engineer —
ის, ვინც ქმნის

Software Engineer ქმნის პროგრამულ სისტემებს — ისეთებს, რომლებიც არა მხოლოდ მუშაობს, არამედ ხარისხსაც ინარჩუნებს და იზრდება.

«როგორ ავაშენოთ სისტემა ისე, რომ მომხმარებელი კმაყოფილი დარჩეს, კოდი სხვა პროგრამისტმაც გაიგოს, და ხვალ — ვითარდებოდეს?»
🏗️

აშენებ და ინარჩუნებ

ყოველდღიურად იღებ ინჟინრულ გადაწყვეტილებებს, სადაც სრულყოფილი პასუხი ყოველთვის არ არსებობს.

🤝

მუშაობ გუნდში

კოდი ერთობლივი პასუხისმგებლობაა. Code Review, PR-ები, Agile — ეს Software Engineering კულტურაა.

🔭

ფიქრობ კონტექსტით

ერთი და იგივე გადაწყვეტილება სხვადასხვა პროექტში სხვადასხვა პასუხისმგებლობაზე გადის.

⚖️

აბალანსებ კომპრომისებს

სიჩქარე ↔ ხარისხი. სიმარტივე ↔ მოქნილობა. ეს Software Engineering ნამდვილი გამოწვევაა.

რეალური კომპრომისები

Software Engineering ყოველდღიური გადასაჭრელი

სიჩქარე ↔ ხარისხი
«გამოუშვი სწრაფად» vs «გამოუშვი სწორად»

ბაზარი სიჩქარეს ითხოვს. სწრაფი გამოშვება ხარისხობრივ დავალიანებას ტოვებს, რომელიც შემდეგ ორმაგი ძალისხმევით სწორდება.

სიმარტივე ↔ მოქნილობა
მარტივი სისტემა vs ყველაფრისთვის მზა სისტემა

მარტივი — ადვილად გასაგებია, ახალ მოთხოვნებს ვერ ეგუება. მოქნილი — ყველაფერს ეგუება, მაგრამ რთულია.

ახლა ↔ მომავალი
«ახლა გამოვიდეს» vs «მომავალში გაფართოვდეს»

ვინც ყოველთვის «მომავლისთვის» წერს — ნელია. ვინც მხოლოდ «ახლანდელისთვის» — მოვალეა.
